# Team Roles & Task Division

This document outlines the four main roles required to complete the Predictive and Interactive Process Scheduler project. Team members can assign their names to the role that best fits their skills and interests.

## 1. 🟢 The Telemetry Engineer (Java Core)
**Owner:** `[Unassigned]`
**Focus Phase:** Phase 1 (Data Gathering)
**Responsibilities:**
- Hook into the core OS classes (`SystemOS.java`, `Process.java`, `CPU.java`).
- Implement `TelemetryManager.java` to count CPU bursts and I/O blocks in $O(1)$ time complexity.
- Ensure the telemetry data is formatted and written correctly to a CSV file without causing bottlenecks in the Java simulator.
**Recommended Skills:** Strong Java, good understanding of OS states and queues.

## 2. 🧠 The Data Scientist (ML Engineer)
**Owner:** `Mariana Sandoval`
**Focus Phase:** Phase 2 (Machine Learning)
**Responsibilities:**
- Work exclusively in the `ml_training/` Python directory.
- Ingest the CSV datasets generated by the Telemetry Engineer using `pandas`.
- Train a Decision Tree classifier using `scikit-learn` to predict if a process should be prioritized based on the User Profile.
- Export the trained model's logic into a lightweight, easily readable format (e.g., auto-generated `if-else` Java code, or JSON) for Phase 3.
**Recommended Skills:** Python, Pandas, Scikit-learn, Data Analysis.

## 3. ⚙️ The Scheduler Architect (Java ML Integrator)
**Owner:** `[Unassigned]`
**Focus Phase:** Phase 3 (Predictive Planning)
**Responsibilities:**
- Create the `ML_Scheduler.java` class inside the `predictive` package.
- Parse and integrate the ML rules generated by the Data Scientist into the Java Scheduler.
- Ensure that the scheduler can read the "User Intent" variable globally and apply the ML logic in milliseconds during the Context Switch.
**Recommended Skills:** Advanced Java, algorithms, understanding of CPU scheduling (Round Robin, FCFS).

## 4. 📊 The QA & Benchmarking Engineer (UI & Analytics)
**Owner:** `[Unassigned]`
**Focus Phase:** Phase 1 (UI) & Phase 4 (Evaluation)
**Responsibilities:**
- Implement the `UserIntentInterface.java` that prompts the user for their profile (Development, Office, Multimedia) during the UR-OS boot sequence.
- Design test scenarios to run the simulator under heavy loads.
- Compare the new `ML_Scheduler` against the classic algorithms.
- Generate visual graphs (Response Time vs. Resource Utilization) demonstrating the project's success.
**Recommended Skills:** Java (for the boot UI), Python/Matplotlib/Excel (for plotting benchmarking graphs), analytical skills.
